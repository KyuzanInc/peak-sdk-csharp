# Security policy

## Supported versions

The `1.0.x` line is the supported stable line for `KyuzanInc.Peak.Sdk`.

## Reporting a vulnerability

Do not open a public issue for a suspected vulnerability. Use GitHub's private
vulnerability reporting for
[`KyuzanInc/peak-sdk-csharp`](https://github.com/KyuzanInc/peak-sdk-csharp/security/advisories/new)
instead. We acknowledge reports within five business days and coordinate a
private disclosure and remediation plan with the reporter.

Never include project API keys, session JWTs, signing keys, private keys, or
private package tokens in a report. If a credential may have been exposed,
revoke it before submitting the report and describe only the impact and the
steps needed to reproduce the issue safely.
