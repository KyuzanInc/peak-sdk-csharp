## What this repo is

A .NET NuGet port of the Unity-only `peak-sdk-unity` SDK. The
cryptographic core is consumed from the external
[`KyuzanInc.Turnkey.Sdk`](https://github.com/KyuzanInc/turnkey-sdk-csharp)
NuGet package (shipped via GitHub Packages, pinned exact in
`Directory.Packages.props` plus the committed `packages.lock.json`
files under `packages/peak-sdk-csharp/{src,tests}/`).

## What this repo is NOT

- It is NOT the canonical source of the Peak public OpenAPI spec. That
  lives in the `KyuzanInc/peak` monorepo. We sync the spec into
  `upstream-snapshots/peak-server-openapi/`, tracking `main` HEAD recorded
  as an exact commit in `PIN.md`.
- It is NOT the source of the Turnkey crypto port. Crypto code, its
  pinned upstream sources, and the multi-round Codex review evidence
  all live in `KyuzanInc/turnkey-sdk-csharp`. Open a PR there for any
  crypto change; the review-evidence gate runs in that repo's CI.

## Cryptographic code

Crypto lives in `KyuzanInc/turnkey-sdk-csharp`. It is consumed from
this repo via `<PackageReference Include="KyuzanInc.Turnkey.Sdk" />`.
Do **not** vendor a copy of any file from that package back into this
repo. If a behaviour change is needed:

1. Open a PR against `KyuzanInc/turnkey-sdk-csharp` with the algorithm
   change and the upstream TypeScript citation.
2. Land it under that repo's multi-round Codex review gate.
3. Cut a new `KyuzanInc.Turnkey.Sdk` release.
4. Bump the pin in `Directory.Packages.props` here, refresh the
   `packages.lock.json` files, and run the wire-format smoke
   (`packages/peak-sdk-csharp/tests/TurnkeyWireFormatSmokeTests.cs`).
5. Update `docs/sync-rules.md` if the bump procedure itself changed.

## Auto-generated code

These files are generated; never hand-edit them:

- `packages/peak-public-api-client-csharp/src/**/*` — OpenAPI codegen
- `upstream-snapshots/peak-server-openapi/public-api.yaml` — synced from
  `KyuzanInc/peak` `main` HEAD, recorded as an exact commit (see
  [docs/sync-rules.md](docs/sync-rules.md))
- `upstream-snapshots/peak-sdk-unity/` — pinned port source
- `packages/peak-sdk-csharp/src/packages.lock.json`,
  `packages/peak-sdk-csharp/tests/packages.lock.json`, and
  `packages/peak-public-api-client-csharp/packages.lock.json` — generated
  by `dotnet restore`; commit alongside any `Directory.Packages.props`
  change.
- `packages/peak-public-api-client-csharp/src/**/*` — generated by
  `scripts/generate-public-api-client.sh` from the pinned
  `upstream-snapshots/peak-server-openapi/public-api.yaml`; never
  hand-edit (drift CI regenerates and diffs).

To resync, follow the procedure in [docs/sync-rules.md](docs/sync-rules.md).

## Storage and secure-storage code (`packages/peak-sdk-csharp/src/Storage/`)

Wallet SDK security policy:

- `InMemoryStorage` is the only default. Anything persistent must be
  explicit opt-in.
- `UnsafePlaintextPlayerPrefsStorage` (in the Unity adapter) requires a
  compile symbol (`PEAK_UNSAFE_STORAGE_OPT_IN`) **and** an explicit
  per-instance `acknowledgePlaintext: true` argument.
- `DpapiSecureStorage` is Windows-only. Other platforms have no built-in
  secure storage in v0.1.0; `ISecureStorage.IsAvailable` returns `false`.
  Consumers MUST handle that case. The threat model
  ([docs/security/storage-threat-model.md](docs/security/storage-threat-model.md))
  is normative.

## Working with this repo

Do not introduce dependencies on:

- `UnityEngine` in any **shipped** SDK package (everything under `packages/`).
  The Unity adapter `packages/peak-sdk-csharp-unity/` is the only package that
  may. Consumer **samples** under `examples/**` (e.g.
  `examples/peak-sdk-unity-reference/`) are NOT shipped packages and are not in
  `peak-sdk-csharp.sln`; they may reference `UnityEngine`. The dependency arrow
  stays example→SDK — no SDK package may reference an example.
- A new logger abstraction (use `Microsoft.Extensions.Logging.Abstractions`
  and `NullLogger<T>.Instance` as default)

Before committing, verify (locally, with GitHub Packages auth set up
per [docs/development.md](docs/development.md)):

```bash
dotnet restore peak-sdk-csharp.sln --locked-mode
dotnet build   peak-sdk-csharp.sln -c Release
dotnet test    peak-sdk-csharp.sln -c Release --filter "Category!=E2E"
```

is green.

## Branch and commit conventions

- `main` is the default branch (no `develop` separate from `main` here).
- Commit message prefix: `feat`, `fix`, `chore`, `docs`, `test`, `refactor`,
  `port` (for upstream port commits).
- For a `KyuzanInc.Turnkey.Sdk` version bump, link the upstream release
  notes in the commit body and include the refreshed
  `packages.lock.json` files in the same commit.
